Hooks

A hook is a VI that is executed either before (pre) or after (post) a Eulogy-related action, which enable Eulogy users to customize and/or extend the conventions and automations used in the Eulogy add-on and toolkit. Eulogy-related actions include:

  • Bury - remove a grave from the target VI's block diagram.
  • Dig - create a grave around a selection of GObjects in a target VI's block diagram.
  • Reanimate - create a zombie from the corpse case in a grave.
  • Smite - disable the corpse case with a zombie in a grave.

For each of these actions, a hook VI is executed regardless if the action was initated from the Shortcut Menu Plugin (SMP), the Graveyard, Quick Drop Keyboard Shortcut (QDKS) plugin, or toolkit API.

A hook VI must have the appropriate name, connector pane pattern, connected terminals, and be located in a specific location in order to be executed before or after a specific Eulogy-related action. The naming convention is a prefixed "Pre" or "Post" followed by a hyphen, "-", then the action name: "Bury", "Dig", "Reanimate", or "Smite", and ending with the file extension, ".vi". For example, the hook to be executed before a Smite action would be, "Pre-Smite.vi", while a hook to be executed after a Dig action would be, "Post-Dig.vi". Thus, there are the following supported hook names:

  • Pre-Bury.vi - executed before a corpse, or zombie, is buried within a target VI's block diagram.
  • Post-Bury.vi - executed after a corpse, or zombie, is buried within a target VI's block diagram.
  • Pre-Dig.vi - executed before a grave is created within a target VI's block diagram.
  • Post-Dig.vi - executed after a grave is created within a target VI's block diagram.
  • Pre-Reanimate.vi - executed before the corpse case within a grave is converted to a zombie within a target VI's block diagram.
  • Post-Reanimate.vi - executed after the corpse case within a grave is converted to a zombie within a target VI's block diagram.
  • Pre-Smite.vi - executed before a zombie within a grave is neutralized within a target VI's block diagram.
  • Post-Smite.vi - executed after a zombie within a grave is neutralized within a target VI's block diagram.

When a Eulogy-related action is executed, regardless of origin, the corresponding action will look for a Pre (before) hook VI, execute the pre-hook VI if found, execute the action, look for a Post (after) hook VI, and finally execute the post-hook VI if found. The action looks for both the Pre and Post hooks in the following, priority-ordered locations (highest priority listed first):

  1. [Application Directory]\Eulogy\Hooks
  2. [Application Directory]\.eulogy\Hooks
  3. [LabVIEW Data Directory]\Field R&D Services\Eulogy\Hooks
  4. [vi.lib]\Field R&D Services\Eulogy\Hooks

This means that only the hook VI at the highest priority location will be executed if it meets the previously mentioned criteria (file name convention, connector pane pattern, and correct connected terminals). Hooks located at lower priority directories will not be executed and ignored.

The recommended process for creating a hook VI is to use the Create a Hook... action from Eulogy's Tool Menu add-on, but templates VIs for each hook to be copied to the appropriate location and modified to extend Eulogy are located at [vi.lib]\Field R&D Services\Eulogy\Toolkit\Support. The templates contain the "Template" suffix in the file stem. Thus, for each hook, the following template VI exists:

  • [vi.lib]\Field R&D Services\Eulogy\Toolkit\Support\Pre-Bury Template.vi
  • [vi.lib]\Field R&D Services\Eulogy\Toolkit\Support\Post-Bury Template.vi
  • [vi.lib]\Field R&D Services\Eulogy\Toolkit\Support\Pre-Dig Template.vi
  • [vi.lib]\Field R&D Services\Eulogy\Toolkit\Support\Post-Dig Template.vi
  • [vi.lib]\Field R&D Services\Eulogy\Toolkit\Support\Pre-Reanimate Template.vi
  • [vi.lib]\Field R&D Services\Eulogy\Toolkit\Support\Post-Reanimate Template.vi
  • [vi.lib]\Field R&D Services\Eulogy\Toolkit\Support\Pre-Smite Template.vi
  • [vi.lib]\Field R&D Services\Eulogy\Toolkit\Support\Post-Smite Template.vi

Be sure to copy the file and rename it without the "Template" suffix, i.e. the Pre-Bury Template.vi copy should be renamed to Pre-Bury.vi; otherwise, the corresponding action will not find and execute the hook VI.
